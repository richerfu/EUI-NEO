# 平台能力

当前平台能力位于 `core/platform.h` / `core/platform.cpp`。这里放跨平台、非业务的系统集成能力，例如打开 URL、文件选择对话框、托盘、IME 光标位置同步。

`core/` 中的 API 命名必须保持通用。文件选择只表达“选择文件”，不要在核心层写成媒体、图片、配置文件等业务概念；具体业务类型由调用方传入选项。

## 打开 URL

```cpp
core::platform::openUrl("https://github.com/sudoevolve/EUI-NEO");
```

平台实现：

- Windows：`ShellExecuteA`
- macOS：`open`
- Linux：`xdg-open`

返回 `true` 表示系统接受了打开请求，不代表目标 URL 一定加载成功。

## 文件选择

推荐使用 `openFileDialog(...)`，它能区分选中、取消和失败：

```cpp
const core::platform::FileDialogResult result =
    core::platform::openFileDialog({
        "Select image",
        {".png", ".jpg", ".jpeg", ".webp"},
        "assets",
        "Images"
    });

if (result.selected()) {
    const std::string path = result.paths.front();
}
```

当前公开 API：

```cpp
struct FileDialogOptions {
    std::string prompt;
    std::vector<std::string> allowedExtensions;
    std::string initialDirectory;
    std::string filterName;
    bool allowMultiple = false;
};

enum class FileDialogStatus {
    Selected,
    Cancelled,
    Failed
};

struct FileDialogResult {
    FileDialogStatus status = FileDialogStatus::Cancelled;
    std::vector<std::string> paths;
    std::string error;

    bool selected() const;
};

core::platform::FileDialogResult openFileDialog(const FileDialogOptions& options = {});
std::string chooseFile(const FileDialogOptions& options = {});
std::vector<std::string> chooseFiles(const FileDialogOptions& options = {});
```

### 选项说明

- `prompt`：对话框标题或提示文本。为空时使用默认提示。
- `allowedExtensions`：允许选择的扩展名列表。可以写 `"png"`、`".png"` 或 `"*.png"`；内部会统一转换。为空表示不过滤。
- `initialDirectory`：初始目录。为空时由系统决定默认位置。
- `filterName`：过滤器显示名称，例如 `"Images"`、`"Text files"`。为空时使用通用名称。
- `allowMultiple`：是否允许多选。`chooseFiles(...)` 会自动打开多选；`chooseFile(...)` 会强制单选。

### 类型过滤

类型过滤是扩展名过滤，不做文件内容探测。例如：

```cpp
core::platform::FileDialogOptions options;
options.prompt = "Select data file";
options.allowedExtensions = {".json", ".csv", ".txt"};
options.filterName = "Data files";
```

平台转换方式：

- Windows：原生 `GetOpenFileNameW` 过滤器，例如 `*.json;*.csv;*.txt`。
- macOS：AppleScript `choose file of type {"json", "csv", "txt"}`。
- Linux：`zenity --file-filter`，不可用时回退到 `kdialog` 过滤器。

### 简单入口

不需要区分取消和失败时，可以使用兼容型入口：

```cpp
const std::string path = core::platform::chooseFile({
    "Select file",
    {".txt", ".md"}
});

if (!path.empty()) {
    // user selected a file
}
```

多选：

```cpp
const std::vector<std::string> paths = core::platform::chooseFiles({
    "Select files",
    {".png", ".jpg"}
});
```

`chooseFile(...)` 和 `chooseFiles(...)` 在用户取消或平台失败时都会返回空结果。需要展示错误信息时，使用 `openFileDialog(...)` 并检查 `result.status` / `result.error`。

### 结果处理

```cpp
const auto result = core::platform::openFileDialog({
    "Select file",
    {".json"}
});

if (result.status == core::platform::FileDialogStatus::Selected) {
    // result.paths
} else if (result.status == core::platform::FileDialogStatus::Cancelled) {
    // user cancelled; usually no UI error needed
} else {
    // result.error contains a diagnostic message
}
```

## 平台依赖

- Windows：使用系统 Common Dialog，不需要额外运行时工具。
- macOS：使用系统 `osascript` 调用 `choose file`。
- Linux：优先使用 `zenity`，失败时尝试 `kdialog`。两者都不可用时，`openFileDialog(...)` 返回 `FileDialogStatus::Failed`。

## 当前限制

- 目前只支持打开本地文件。
- 暂不支持选择目录、保存文件、默认文件名、父窗口/modal 绑定。
- 文件选择是阻塞式系统对话框。适合在按钮点击回调里调用；不要在每帧 compose/update 中调用。
